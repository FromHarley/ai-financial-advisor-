# `bias_audit_chart.png` — Reference for Claude Code

## What this file is

`bias_audit_chart.png` is the **visual artifact of the fairness audit**. It's a stacked bar chart showing how the trained ML model distributes predicted risk tiers across age bands. The chart is the most legible piece of the entire Responsible AI layer — anyone, technical or non-technical, can look at it and immediately see the pattern the audit uncovered.

## Where it lives

`layer4_respai/bias_audit_chart.png`

## How it was made

This file is **generated by `bias_audit.py`**. It's not hand-drawn or designed in a separate tool. When you run:

```
python -m layer4_respai.bias_audit
```

the script:

1. Runs the trained Random Forest classifier across all 2,000 training profiles.
2. Computes the share of each predicted tier (Low / Medium / High) within each age band (Under 30, 30–54, 55–69, 70+).
3. Plots a stacked bar chart using matplotlib.
4. Saves the chart at 150 DPI to this file path, overwriting any previous version.

If you ever change the model or the data, **re-run the audit script** to regenerate this chart. Don't edit the PNG directly.

## What the chart shows

- **X-axis:** Four age bands (Under 30, 30–54, 55–69, 70+) using CFPB-standard bins.
- **Y-axis:** Share of profiles in each band, as a percentage from 0 to 100.
- **Three colored segments per bar:**
  - 🟢 **Green = Low risk tier**
  - 🟠 **Orange = Medium risk tier**
  - 🔴 **Red = High risk tier**
- **Title:** "Predicted risk-tier distribution by age band"
- **Legend:** Bottom-right, identifying the three tier colors.

Each bar adds up to 100% — the segments show how the band gets divided across tiers.

## What the chart reveals (summary)

Reading the chart left-to-right:

- **Under 30 bar:** mostly red and orange. About 54% of Under-30 profiles get the High tier. Almost no Low.
- **30–54 bar:** majority orange. About 67% Medium, with the Low/High share roughly balanced (8% Low, 25% High).
- **55–69 bar:** heavy orange with a meaningful green chunk. About 70% Medium, 26% Low, only 4% High.
- **70+ bar:** **empty.** No profiles in the dataset for this band. This is a real gap, not a model output — and it's called out explicitly in the findings text and the model card.

The visual pattern: the red shrinks left-to-right and the green grows. Younger people get pushed toward higher risk; older people get pushed toward lower risk. The model is reproducing the relationship between age and investment horizon that exists in the source labels.

## Why it matters in the project

The course rubric requires a fairness audit. A chart is the right format because:

1. **It's defensible.** A grader can look at the chart and immediately see whether the model's behavior across groups is reasonable or alarming.
2. **It's reproducible.** The chart was generated from a script that anyone with the repo can re-run.
3. **It's honest.** The empty 70+ bar is visible — we didn't paper over the dataset gap.
4. **It pairs with text.** The chart visualizes what `bias_audit_findings.txt` describes in words. Two formats, same conclusion.

## Where the chart appears

- **In the file system:** `layer4_respai/bias_audit_chart.png`
- **In the Streamlit app:** at the bottom of the page, inside the expandable **"📋 Responsible AI — Model Card & Bias Audit"** section. The function `render_bias_audit_summary()` in `bias_audit.py` reads this PNG and embeds it in the page using `st.image()`.
- **In the model card:** referenced from `model_card.md` Section 5 (Bias audit).

## Limitations the chart can't show

The chart is honest about *what the model does*, but it can't on its own answer *whether that's fair*. Specifically:

- It can't show disparate impact on protected classes (gender, race, education) because the training data doesn't include those columns.
- It can't distinguish "the model is biased" from "the labels are biased and the model is reproducing them."
- It treats age as a proxy for demographics, which it isn't — age is correlated with many demographic factors but isn't equivalent to any of them.

These caveats are spelled out in `bias_audit_findings.txt` and in the model card. The chart visualizes the data — the text contextualizes it.

## How to talk about this in the presentation

Show the chart. It's the most visual artifact in the project. The talk track:

*"This is what the model does across age groups. Younger profiles skew red — high risk. Older profiles skew green — low risk. The 70+ band is empty because the training dataset has no 70+ profiles, which is a gap we call out openly. The pattern itself isn't surprising — younger investors with longer time horizons are typically advised to take on more risk — but documenting it visually is what makes the model's behavior accountable."*
